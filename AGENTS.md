# AGENTS.md

## Project goals
- Maintain a fast, static personal website hosted on GitHub Pages.
- Prioritize readability, accessibility, and simple deployment.

## Tech rules
- Use plain HTML, CSS, and vanilla JavaScript unless explicitly told otherwise.
- Do not introduce a framework, package manager, or build step without justification.
- Keep all assets organized under /assets.

## Design rules
- Clean, minimal layout
- Mobile-first
- Good contrast
- No auto-playing media
- Avoid heavy animations

## Definition of done
- Summarize commands run and their outputs in the final handoff.
- No unrelated refactors in the same change.
- New behavior is documented.

## Review expectations
- Preserve GitHub Pages compatibility.
- Do not break the custom domain setup.
- Do not rename files casually.
- Keep CSS simple and documented.
- Treat broken links, accessibility regressions, and layout breakage as high priority.
- State what changed, why it changed, and what evidence proves it.
- Call out any skipped checks or assumptions explicitly.
